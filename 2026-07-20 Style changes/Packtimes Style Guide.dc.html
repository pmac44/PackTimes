Style Guide

Two themes — Paper (day) and Graphite (night). Same layout, same rules; only tokens change. Implementation: tokens.css in this project, switched via <html data-theme="paper|graphite">.

1 · Color

One surface family per theme; hierarchy comes from raised/sunken steps and hairlines, never from new hues. Green is demoted from “text color” to accent: it marks the active tab, progress, and primary values only.

Standard hue semantics (grey→blue→green→amber→red→purple), muted to sit in the palette. A zoned cell paints its top half in the zone color with white digits; non-zoned cells keep the dark top. Never use warm-deep amber on power in a zoned context — false Z4 read.

2 · Typography

Mono for physics, proportional for accumulation. Values that churn every second (speed, power, cadence, HR) use fixed-width digits so they don't jump; values that sit still (distance, time, stoppage, ETA) use the warmer proportional face.

  • Cell digits: 36px primary / 30px secondary (head-unit legible at handlebar distance).
  • Values over 5 characters step down one size (36→30). Only distance triggers this; alternatively drop the decimal above 1,000 km (1235 km) and never shrink.
  • Labels 11px, always below the number, always muted. Units ride the baseline at ~40% of digit size.
  • Worst-case widths to test: 99.9 speed · 1234.5 distance · 111.1h time · 4-digit watts.

3 · Paired data cells

  • Two-tone: dark top = live value, light bottom = paired/average. The B&W split is the legibility feature — it survives both themes (bottom stays light even in Graphite).
  • All 6 cell locations identical size — cells are interchangeable; tap cycles the metric, one state is blank (dashed outline, “+”).
  • Floating over the map: 1.5px border + shadow, ~95% opaque. OSM tiles are a fixed given — chrome must win contrast against them, not assume a styled map.
  • Zone-capable cells (power, HR) tint the top half with the zone color.

4 · Layout rules

  • Chevron progress bar stays — signature element, full-width under the tab bar.
  • Progress row: distance-to-go (accent, left) · ridden (small, center) · total (right), 36px Space Grotesk.
  • Tab bar: icon + short label, 6 tabs, full-column hit areas ≥ 44px. Never icon-only.
  • Map controls grouped in one vertical rail (Komoot pattern), positioned below the cell grid — never scattered circles.
  • Bottom stack: elevation strip (with position marker + day-split band) above FOOD/SLEEP next-stop rows.
  • Hit targets ≥ 44px everywhere; primary ride actions bigger (gloved, one-handed).
  • Max two surface colors per screen; hierarchy via raised/sunken + hairlines, not new colors.

5 · Night & battery

Graphite's sunken surface (#101216) is near-black for OLED savings; a "pure black" variant (surface → #000) is a one-token change if you want maximum battery mode. Auto-switch themes on sunset or ambient light; the toggle lives in Settings and writes data-theme.
